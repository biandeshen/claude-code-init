## 一、Claude Code 内置工具速查

| 工具 | 典型用法 |
|------|----------|
| `Bash` | `git log`、`pytest`、`ruff` |
| `Read` / `Edit` / `Write` | 读写代码 |
| `Grep` / `Glob` | 搜索引用、找文件 |
| `Task` | 复杂任务分解跟踪 |
| `WebFetch` / `WebSearch` | 获取网页内容、搜索 |

## 二、常用工作流模式

### 2.1 代码审查工作流

```
Read(目标文件) → Bash(git log -p -3) → Grep(检查引用) → Task(分析报告)
```

**场景**：用户要求审查代码
**执行步骤**：
1. `Read` 读取目标文件完整内容
2. `Bash: git log -p -3 -- <file>` 分析历史变更
3. `Grep` 检查相关引用和依赖
4. `Task` 启动子代理进行深入分析
5. 输出审查报告

### 2.2 Bug 修复工作流

```
Read(错误上下文) → Grep(定位问题) → Bash(pytest) → [假设] → Edit(修复) → Bash(验证)
```

**场景**：用户报告 bug
**执行步骤**：
1. `Read` 读取报错文件或日志
2. `Grep` 定位问题代码位置
3. `Bash: pytest` 或 `ruff check` 复现错误
4. `[假设]` 输出假设并记录到 Plan.md
5. `Edit` 进行最小化修复
6. `Bash: pytest` 验证修复

### 2.3 重构工作流

```
Task(分解任务) → Read(理解结构) → Grep(检查依赖) → [假设] → Edit/Write(修改) → Bash(测试)
```

**场景**：需要重构代码
**执行步骤**：
1. `Task` 分解大任务为子任务
2. `Read` 理解当前代码结构
3. `Grep` 检查所有引用点
4. `[假设]` 输出重构方案假设
5. `Edit` 逐步修改或 `Write` 创建新实现
6. `Bash: pytest` + `ruff check` 验证

### 2.4 文档编写工作流

```
Glob(找模板) → Read(参考) → Write(新建) → Bash(git add)
```

**场景**：编写文档
**执行步骤**：
1. `Glob` 查找相关模板或参考文档
2. `Read` 读取模板格式
3. `Write` 按照模板格式编写
4. `Bash: git add` 添加到版本控制

## 三、主动升级模式

以下情况立即暂停，请求确认：

| 触发条件 | 动作 |
|----------|------|
| 提交含 `!important`/`hack`/`fix` | `Bash: git log -p` 检查 |
| 影响超过 3 个组件 | `Grep` 检查引用 |
| 不可逆操作 | 请求确认 |
| 假设被否定 | 立即停止，重新分析 |
