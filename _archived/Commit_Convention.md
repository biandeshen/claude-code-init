# 提交规范

> 版本：v1.0.0 | 更新：2026-04-21
> 本文档描述 Git 提交信息的规范，适用于任何项目。

---

## 一、提交格式

```
<type>(<scope>): <subject>

[可选 body]

[可选 footer]
```

### 示例

```
feat(auth): 添加用户登录功能

- 实现 JWT 令牌验证
- 添加 refresh token 支持
- 更新用户文档

Closes #123
```

---

## 二、Type 类型

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档更新 |
| `style` | 代码格式调整（不影响功能） |
| `refactor` | 重构（无功能变化） |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具/依赖相关 |

---

## 三、Scope 范围

Scope 表示影响的模块，可根据项目调整：

| 常见 Scope | 说明 |
|------------|------|
| `core` | 核心引擎 |
| `auth` | 认证模块 |
| `api` | API 相关 |
| `ui` / `frontend` | 前端界面 |
| `db` / `database` | 数据库相关 |
| `ci` | CI/CD 相关 |

> 项目可自定义 Scope 列表。

---

## 四、Subject 规则

| 规则 | 说明 |
|------|------|
| 简短 | 不超过 50 字符 |
| 中文 | 使用中文描述 |
| 动词开头 | 用动词描述动作 |
| 不加句号 | 结尾不要句号 |

### 正确/错误示例

```
✅ feat(auth): 添加用户登录功能
✅ fix(core): 修复事件总线内存泄漏
✅ docs: 更新 README

❌ feat(auth): 添加了用户登录功能（多了"了"）
❌ fix: 修复问题（缺少 scope）
❌ feat: Update user login.（英文+太长）
```

---

## 五、Body 规则

- 使用中文
- 每行不超过 80 字符
- 说明 **为什么** 而不只是 **做了什么**

### 示例

```
feat(ui): 添加深色主题支持

用户反馈长时间使用眼睛疲劳，需要暗色模式。
采用 CSS 变量实现，便于后续扩展主题。
```

---

## 六、Footer 规则

用于关联 Issue 或关闭任务：

| 格式 | 说明 |
|------|------|
| `Closes #123` | 关闭 Issue #123 |
| `Refs #456` | 关联 Issue #456 |
| ` BREAKING CHANGE:` | 破坏性变更说明 |

---

## 七、提交前检查

```
- [ ] 所有修改文件已保存
- [ ] 代码格式正确（无 lint 错误）
- [ ] 有对应的测试（如果有）
- [ ] 提交信息符合规范
- [ ] 原子提交（一个提交一个功能点）
```

---

## 八、原子提交原则

### 定义

一个提交只包含**一个可独立验证的功能点**。

### 正确做法

```
# 提交 1：添加登录功能
feat(auth): 添加用户登录功能

# 提交 2：修复样式问题（与登录无关）
fix(ui): 修复按钮样式
```

### 错误做法

```
# 提交 1：登录功能 + 修复样式 + 更新文档 混在一起
feat: 添加登录功能和修复样式和更新文档
```
