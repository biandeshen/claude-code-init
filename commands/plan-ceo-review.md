# /plan-ceo-review - 产品级需求审查

> **依赖声明**：本命令依赖 ECC (Everything Claude Code) 插件。未安装时请使用本地替代方案。
> 本地替代：`/brainstorming` 进行需求分析与头脑风暴。

在编写任何代码行之前，强制挑战需求的合理性。

## 触发条件

- 开始新功能前不确定"该构建什么"
- 收到模糊或不完整的需求
- 需要重新定义产品范围

## 使用方式

```
/plan-ceo-review <需求描述>

示例：
/plan-ceo-review 用户想要一个聊天功能
/plan-ceo-review $ARGUMENTS
```

## 审查流程

### 1. 复述理解
复述你理解的需求，确认理解正确。

### 2. 替代方案
提出 2-3 个可能被忽略的替代方案：
- 更简单的解决方案
- 更复杂的解决方案
- 完全不同的思路

### 3. 投资回报评估
用 1-10 分评估此功能的投资回报率：

| 分数 | 含义 |
|:----:|------|
| 1-3 | 低价值，可能不值得做 |
| 4-6 | 中等价值，按优先级处理 |
| 7-10 | 高价值，优先实现 |

### 4. 范围决策
提供四种范围调整选项：

| 选项 | 描述 |
|------|------|
| 扩展 | 增加范围，提高价值 |
| 选择性扩展 | 部分扩展，只做高价值部分 |
| 保持范围 | 按原计划实现 |
| 缩减 | 缩小范围，降低成本 |

## 输出格式

```markdown
# 需求审查报告

## 需求理解
[你的理解]

## 替代方案
1. [方案1]
2. [方案2]

## 投资回报
评分: X/10
理由: [原因]

## 范围决策
[扩展/选择性扩展/保持/缩减]

## 下一步
[确认后进入技术规划]
```

## 决策门槛

只有在用户确认后才进入技术规划阶段。
