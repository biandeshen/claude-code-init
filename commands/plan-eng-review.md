# /plan-eng-review - 技术架构评审

对技术方案进行多维度架构评审（可扩展性、安全性、可维护性、成本）。

> 由 Router 在用户提到"架构评审"、"eng-review"时自动路由

## 用法

```
/plan-eng-review <方案描述或文件路径>
```

## 执行步骤

1. **方案解读**：理解方案的核心设计、关键决策和约束条件
2. **多维度评审**：
   - 可扩展性：是否支持预期增长、水平扩展路径
   - 安全性：攻击面分析、数据保护策略
   - 可维护性：模块解耦程度、测试策略
   - 成本：基础设施成本、维护人力成本
3. **输出评审报告**：分维度打分 + 风险清单 + 改进建议

## 输出格式

```markdown
## 架构评审报告

| 维度 | 评分 | 说明 |
|------|:----:|------|
| 可扩展性 | X/10 | ... |
| 安全性 | X/10 | ... |
| 可维护性 | X/10 | ... |
| 成本 | X/10 | ... |

### 关键风险
1. ...
2. ...

### 改进建议
#### 短期（本迭代）
- ...
#### 长期（下阶段）
- ...
```

## 注意事项

- 评审聚焦架构层面，不涉及代码细节
- 如有数据不明确，标记为"待确认"而非跳过
