# /refactor - 安全重构

对指定代码进行安全、可验证的重构，不改变外部行为。

## 重构流程

### 1. 确定边界
- 明确重构目标（函数、类、模块）及其对外接口
- 查找所有调用方和依赖项

### 2. 保护性测试
- 为重构目标编写/补充单元测试（覆盖其全部公共接口和关键分支）
- 确认所有现有测试通过，建立安全网

### 3. 执行重构
- 逐步进行，每小步保持测试绿灯
- 允许的重构手法：提取函数/类、内联临时变量、简化条件表达式、重命名、移动方法
- 禁止：改变任何公共接口的签名或行为、修改业务逻辑

### 4. 验证
- 运行重构模块的单元测试 + 项目全量测试
- 比较重构前后的接口行为（若可能）

### 5. 提交
- 通过所有验证后，执行 `/commit`
- Commit 格式：`refactor(<scope>): <重构说明>`

## 安全规则
- 测试失败时立即回退
- 绝不混合重构与功能修改
- 重构范围超过 3 个文件时，分多个 commit 提交

## 输出格式
### 重构计划
- 目标
- 影响范围（文件数、调用方数）
- 重构步骤（每步一个检查点）

### 执行结果
- 修改的文件
- 测试结果
- commit 信息
