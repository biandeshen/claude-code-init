# /refactor - 安全重构

对指定代码进行安全、可验证的重构，不改变外部行为。

## 重构流程

### 1. 确定边界
- 明确重构目标（函数、类、模块）及其公共接口
- 定位所有调用方和依赖项

### 2. 保护性测试
- 为重构目标编写或补充单元测试，覆盖全部公共接口和关键分支
- 确保当前所有测试通过，建立安全网

### 3. 执行重构
- 逐步进行，每完成一步立即运行相关测试，保持测试绿灯
- 允许的重构手法：提取函数/类、内联临时变量、简化条件表达式、重命名、移动方法
- **禁止改变公共接口签名、禁止修改业务逻辑**

### 4. 验证
- 运行目标模块的单元测试 + 项目全量测试
- 比较重构前后接口行为（如可能）

### 5. 提交
- 所有验证通过后，调用 `/commit`
- 提交格式：`refactor(<scope>): <重构说明>`

## 安全规则
- 测试失败立即回退，分析原因
- 绝不在一次重构中混合功能修改
- 重构范围超过 3 个文件，拆分为多个提交

## 输出格式

### 重构计划
- 目标
- 影响范围（文件数、调用方数）
- 重构步骤（每步一个检查点）

### 执行结果
- 修改的文件列表
- 测试结果
- 提交的 commit 信息
