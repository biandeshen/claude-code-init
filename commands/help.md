# /help - 可用命令与快捷参考

显示当前 Claude Code 环境中所有可用命令、工作流和关键技能。

## 一、本项目自定义命令

| 命令 | 用途 | 示例 |
|------|------|------|
| `/review` | 代码审查（当前分支 vs main） | `/review` |
| `/commit` | 规范提交（分析 → 生成信息 → pre-commit → 提交） | `/commit` |
| `/architect <方案>` | 架构评审（多维度分析 + 分优先级的建议） | `/architect 把事件总线换成 Redis` |
| `/validate` | 运行所有项目校验脚本（结构/密钥/函数长度/依赖/导入） | `/validate` |
| `/fix` | 自动错误修复（分析 → 根因 → 方案 → 执行 → 验证） | `/fix` |
| `/refactor` | 安全重构（先加保护测试，逐步重构，保持测试绿灯） | `/refactor` |
| `/explain <文件> [函数/行号]` | 深度代码解释（架构/数据流/关键决策/风险） | `/explain agent_loop.py` |
| `/help` | 显示所有可用命令和工作流速查 | `/help` |

## 二、OpenSpec 工作流（复杂功能开发 SDD）

| 命令 | 用途 |
|------|------|
| `/opsx:propose <变更名>` | 创建提案：为什么要做 |
| `/opsx:spec <变更名>` | 编写规格：要实现什么 |
| `/opsx:design <变更名>` | 设计方案：怎么做 |
| `/opsx:task <变更名>` | 拆解任务：原子步骤 |
| `/opsx:check <变更名>` | 质量门禁：完整性检查 |
| `/opsx:apply <变更名>` | 执行实现 |
| `/opsx:archive <变更名>` | 归档完成 |

> **何时使用**：开发复杂功能时按顺序执行（如新技能、闭环引擎重构、跨模块变更）。日常小改动不需要。

## 三、ECC 常用功能速查

ECC 提供 158+ 个 Skill、38+ 个 Agent 和 72+ 个命令，以下是最常用的：

### 编码与开发
- **`/plan`** — 生成开发计划
- **`/bd`** (Before Development) — 制定需求、设计方案、创建组件规格
- **`/impl`** — 严格按 spec 实现
- **`/bob`** — 全能全栈开发者 Agent
- **`/ut`** — 为现有代码生成单元测试

### 代码审查与质量
- **`/review`** — 全面代码审查（若无自定义版本则使用此）
- **`/security-review`** — 安全隐患扫描
- **`/lint`** — 代码规范检查与修复

### 文档生成
- **`/gen-docs`** — 自动生成/更新项目文档
- **`/bdd`** — 生成 README、架构文档等

### 任务与工作流
- **`/sc`** (Smart Continuity) — 跨会话任务连续性管理
- **`/gdd`** (Goal-Driven Development) — 目标驱动开发

### 记忆与上下文
- **`/memory`** — 管理跨会话持久化记忆
- **`/ctx`** — 查看当前上下文使用情况

## 四、Superpowers 核心能力
Superpowers 无需手动调用命令，通过自然语言触发：
- `用 TDD 方式实现这个函数` — 强制红/绿/重构循环
- `子代理驱动开发` — 自动并行执行子任务
- 调试说 `帮我找到这个 bug 的根因` 自动触发根因追踪模式

## 五、cc-discipline 自动保护
三个 Hook 全自动运行，你无需操作：
- 单文件编辑 ≥5 次 → 强制停止
- 调试未完成 → 禁止修改代码
- 错误后惰性修改 → 提醒先分析
