# /qa - 质量保证测试

> **依赖声明**：本命令依赖 ECC (Everything Claude Code) 插件。未安装时请使用本地替代方案。
> 本地替代：手动测试或用 `/review` 进行代码审查。

对指定的功能或页面进行全面质量保证测试。

## 触发条件

- 功能开发完成后需要验证
- 准备发布前需要回归测试
- 发现 bug 后需要复现和验证修复

## 使用方式

```
/qa <目标> [模式]

示例：
/qa 登录页面
/qa 结账流程 full
/qa 用户设置 regression
```

## 测试模式

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| `diff-aware` | 自动检测代码变更 | 默认，节省时间 |
| `full` | 全面测试 | 重要功能发布前 |
| `quick` | 快速冒烟测试 | 紧急验证 |
| `regression` | 回归测试 | 确保修复不破坏其他功能 |

## 测试流程

### 1. 识别验收标准
明确功能必须满足的条件。

### 2. 设计测试用例
覆盖：
- 正常路径
- 边界条件
- 错误处理路径
- 安全边界

### 3. 执行测试
使用真实浏览器自动化执行。

### 4. 输出报告

```markdown
# QA 测试报告

## 测试目标
[URL 或功能描述]

## 测试模式
[模式]

## 测试结果

| 测试项 | 状态 | 说明 |
|--------|:----:|------|
| 登录成功 | ✅ | |
| 密码错误 | ✅ | 提示正确 |
| XSS 注入 | ✅ | 已拦截 |

## 健康评分
总分: 85/100

## 发现问题

### 严重
- [问题描述]

### 轻微
- [问题描述]

## 建议
[改进建议]
```

## 健康评分标准

满分 100 分，按以下维度评分：

| 维度 | 权重 |
|------|:----:|
| 功能正确性 | 30% |
| 错误处理 | 20% |
| 安全性 | 20% |
| 性能 | 15% |
| 可用性 | 15% |
