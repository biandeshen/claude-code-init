# /review - 代码审查命令

对提交的代码进行全面审查。

## 触发条件

当用户输入 `/review` 或说"审查代码"、"帮我 review"时使用。

## 执行步骤

1. **影响面分析**
   - 运行 `git diff --cached` 查看暂存区变更
   - 或 `git diff HEAD~3..HEAD` 查看最近提交
   - 识别高风险变更区域

2. **审查维度**
   - 代码风格一致性
   - 潜在 bug
   - 安全性问题
   - 性能影响
   - 测试覆盖

3. **输出格式**

```markdown
## 代码审查报告

### 概览
- 变更文件: N 个
- 新增行数: +N
- 删除行数: -N

### 高风险问题
...

### 中风险问题
...

### 建议
...

### 结论
✅ 可以合并 / ⚠️ 需要修改后合并 / ❌ 阻止合并
```

## 注意事项

- 如果没有暂存或未跟踪的文件，提示用户先提交或暂存
- 复杂变更建议使用 Superpowers 的 `requesting-code-review` 技能
