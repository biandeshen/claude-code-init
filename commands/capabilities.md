# /capabilities - 展示系统全部能力

按开发阶段索引的能力视图，帮助你找到当前场景最合适的工具。

## 按开发阶段索引

### 📝 需求阶段

| 场景 | 命令/Skill | 说明 |
|------|------------|------|
| 头脑风暴方案 | 说"帮我规划这个功能" | 自动触发 brainstorming Skill |
| 需求审查 | `/plan-ceo-review <需求>` | 产品级需求审视 |
| 架构设计 | `/plan-eng-review <方案>` | 技术架构评审 |

### 💻 编码阶段

| 场景 | 命令/Skill | 说明 |
|------|------------|------|
| 测试驱动开发 | 说"用 TDD 方式实现" | 自动触发 tdd-workflow |
| 安全重构 | 说"重构这个模块" | 自动触发 safe-refactoring |
| 并行开发 | `/team 3 <任务>` | 3 个 Agent 同时工作 |

### 🔍 审查阶段

| 场景 | 命令/Skill | 说明 |
|------|------------|------|
| 代码审查 | 说"审查代码" | 自动触发 code-review |
| 架构评审 | `/architect <方案>` | 架构方案评审 |
| 质量保证 | `/qa <目标>` | 浏览器自动化测试 |

### 🐛 修复阶段

| 场景 | 命令/Skill | 说明 |
|------|------------|------|
| 系统化调试 | 说"排查这个 bug" | 自动触发 error-fix |
| 自动修复 | 说"修复这个错误" | 自动触发 error-fix |

### 📦 提交阶段

| 场景 | 命令/Skill | 说明 |
|------|------------|------|
| 规范提交 | `/commit` | 自动分析变更并生成规范提交 |

### 🌙 无人值守

| 场景 | 命令 | 说明 |
|------|------|------|
| 过夜任务 | `bash scripts/tmux-session.sh` | 让 AI 通宵工作 |
| 查看报告 | `/overnight-report` | 汇总昨晚的工作成果 |
| 状态仪表盘 | `/status` | 项目完整状态概览 |

### 📊 项目校验

| 场景 | 命令 | 说明 |
|------|------|------|
| 运行所有检查 | `/validate` | 执行全部 5 个校验脚本 |
| 依赖检查 | `python scripts/check_dependencies.py` | 检查项目依赖 |

## 常用命令速查

| 命令 | 作用 |
|------|------|
| `/review` | 代码审查 |
| `/fix` | 自动修复 |
| `/commit` | 规范提交 |
| `/refactor` | 安全重构 |
| `/tdd` | 测试驱动开发 |
| `/explain <目标>` | 代码解释 |
| `/validate` | 运行校验 |
| `/team <N> <任务>` | 启动 Agent 团队 |
| `/qa <目标>` | QA 测试 |
| `/status` | 项目状态 |
| `/help` | 查看帮助 |

> 输入 `/help <命令>` 查看详情，`/<命令>` 直接执行。
