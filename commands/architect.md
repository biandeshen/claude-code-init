# /architect - 架构分析与设计

对项目或模块进行架构分析和设计建议。

## 触发条件

当用户输入 `/architect` 或说"分析架构"、"设计建议"时使用。

## 执行步骤

1. **项目结构分析**
   - 读取项目目录结构
   - 识别核心模块和依赖关系

2. **现有架构评估**
   - 模块解耦程度
   - 依赖方向是否正确
   - 是否符合项目特点（语言、规模、团队）

3. **设计建议**
   - 基于问题给出建议
   - 说明优缺点
   - 提供可选方案

4. **输出格式**

```markdown
## 架构分析报告

### 当前结构
```
项目结构图
```

### 依赖关系
```
依赖图
```

### 问题识别
1. ...
2. ...

### 设计建议
#### 方案 A
- 优点: ...
- 缺点: ...
- 适用场景: ...

#### 方案 B
- ...

### 推荐方案
...

### 实施路径
1. ...
2. ...
```

## 注意事项

- 如果没有具体问题，询问用户想要解决什么问题
- 架构决策要考虑团队能力和维护成本
- 建议要有具体的代码示例或结构图
