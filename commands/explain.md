# /explain - 代码解释

解释指定代码的架构、数据流和关键决策。

## 使用方式
- `/explain <文件路径>` — 解释整个文件
- `/explain <文件路径> <函数名>` — 解释特定函数
- `/explain <文件路径> <行号范围>` — 解释特定代码段

## 输出内容

### 1. 概览（一句话）
该代码的核心职责

### 2. 架构定位
- 在项目中的层级（核心层/技能层/工具层）
- 依赖和被依赖关系
- 位置：`<文件路径>`

### 3. 数据流
- 输入（从哪里来）
- 处理（核心逻辑步骤）
- 输出（到哪里去）

### 4. 关键决策
- 为什么选择了这种实现方式（而非最显而易见的替代方案）
- 注明的技术权衡和假设

### 5. 潜在风险
- 已知瓶颈
- 假设前提
- 未来可能的变更方向

## 输出格式
以 Markdown 输出，每个部分用独立段落，代码块标注语言。

示例：
### 概览
`get_user()` 负责从数据库获取用户信息并进行权限校验。

### 架构定位
- 层级：数据访问层
- 被 `AuthMiddleware`、`UserController` 依赖
...
