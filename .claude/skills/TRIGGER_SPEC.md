# Skill 触发词规范 v1.0.0

定义 Skills 的触发词命名和声明规范。

## 格式

- 触发词在 SKILL.md frontmatter 的 `description` 字段中声明
- 中文触发词：用顿号或逗号分隔
- 英文触发词：用逗号分隔
- 分为 `中文触发词` 和 `英文触发词` 两个段落

示例：
```yaml
中文触发词：审查、检查、review、看看代码、检查质量、代码审计
英文触发词：review, check, examine, inspect, audit, quality
```

## 命名原则

1. **长度限制**：每个触发词不超过 8 个字（中文）或 3 个单词（英文）
2. **自然语言**：触发词应为用户自然语言中的常见表达
3. **避免单字**：禁止单字触发词（如 "改"、"查"），防止路由过载
4. **包含变体**：同时包含完整形式（"代码审查"）和口语化形式（"帮我看看"）
5. **中英对照**：每个 Skill 必须同时提供中英文触发词

## 触发词的 Router 协作

1. 触发词声明在 Skill frontmatter 的 `description` 中
2. Router 决策表根据触发词进行语义匹配
3. 否定词过滤在 Router 端执行（不依赖 Skill 声明）
4. 新增触发词时同步更新：
   - Skill 的 `description` frontmatter
   - Router 的决策表

## 禁止事项

- 禁止触发词跨 Skill 重叠（两个 Skill 声明相同的触发词）
- 禁止使用过于宽泛的触发词（如 "帮我"、"看一下"）
- 禁止与实际功能不符的触发词（如声明 "部署" 但无部署能力）
