# 安全重构流程

## 核心原则
- 重构不改变外部行为，只改善内部结构
- 必须先有保护测试，才能执行重构
- 每次小步修改后立即运行测试，保持持续绿灯

## 执行流程

### 第一步：确定边界
1. 明确重构目标（函数、类、模块）
2. 列出所有公共接口、调用方和依赖项
3. 评估影响范围

### 第二步：建立保护网
1. 检查重构目标是否已有完整测试
2. 如果缺失，先补充单元测试：
   - 覆盖所有公共接口
   - 覆盖关键分支和边界条件
3. 确认所有现有测试通过

### 第三步：逐步重构
1. 每次只做一步小的改动
2. 每步改动后立即运行相关测试
3. 保持测试持续绿灯
4. 允许的重构手法：
   - 提取函数/类
   - 内联临时变量
   - 简化条件表达式
   - 重命名变量/函数/类
   - 移动方法到更合适的类

### 第四步：验证
1. 运行重构模块的单元测试 + 项目全量测试
2. 确认没有引入新的错误或警告

### 第五步：提交
- 通过所有验证后执行 `/commit`
- 格式：`refactor(<scope>): <重构说明>`
- 重构范围超过 3 个文件时，分多个 commit 提交
